  1. No HTML qual a diferença entre <article>, <section>, <aside> e <nav>?

    <article>: Representa um conteúdo independente e autocontido, como uma notícia ou post.
    <section>: Define uma seção temática do documento, agrupando conteúdo relacionado.
    <aside>: Indica conteúdo complementar, como barras laterais ou caixas de informação.
    <nav>: Destina-se à navegação, agrupando links de menus ou navegação interna.

  2. O que é Mobile-First Indexing e como ele impacta o ranking dos sites? Explique as melhores práticas para garantir que um site seja otimizado para o Mobile-First Indexing.

    Mobile-First Indexing significa que o Google prioriza a versão mobile do site para indexação e ranking. Sites não otimizados para dispositivos móveis podem perder posições nos resultados de busca.
    Boas práticas: Design responsivo, carregamento rápido, fontes legíveis, botões acessíveis, evitar pop-ups intrusivos e garantir o mesmo conteúdo em desktop e mobile.

  3. O que são dados estruturados (Structured Data) e como eles influenciam o SEO? Cite dois exemplos de tipos de dados estruturados que podem ser usados para melhorar a visibilidade nos resultados de busca do Google.

    Dados estruturados são marcações no código (ex: JSON-LD) que ajudam mecanismos de busca a entender o conteúdo da página. Eles melhoram o SEO ao possibilitar rich snippets.
    Exemplos: FAQPage (perguntas frequentes), Product (produto com preço e avaliações).

  5. Em Javascript qual é a diferença entre métodos definidos diretamente no objeto e métodos definidos no prototype?

    Métodos definidos diretamente no objeto são exclusivos daquela instância, enquanto métodos no prototype são compartilhados entre todas as instâncias, economizando memória e promovendo reutilização.

  6. Explique o que são Hooks no React e qual problema eles vieram resolver. Cite pelo menos três exemplos.

    Hooks são funções que permitem usar estado e outros recursos do React em componentes funcionais. Eles resolvem a limitação de não poder usar estado ou ciclo de vida em funções.
    Exemplos: useState, useEffect, useContext.

  7. O que é uma classe anônima em PHP? Em quais situações ela é utilizada? Crie um exemplo de classe anônima que defina um método e imprima uma saudação.

    Uma classe anônima em PHP é uma classe sem nome, útil para instâncias únicas ou uso pontual.
    Exemplo: 

    $obj = new class {
  public function saudacao() {
    echo "Olá, mundo!";
  }
};
$obj->saudacao();

  8. O que é um JOIN em SQL? Cite e explique dois tipos de JOIN.

    JOIN é uma operação que combina registros de duas ou mais tabelas.
    INNER JOIN: Retorna apenas registros com correspondência em ambas as tabelas.
    LEFT JOIN: Retorna todos os registros da tabela da esquerda e os correspondentes da direita, se existirem.
